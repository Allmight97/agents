# Parallels Windows VM Findings - 2026-07-06

Use this as a dated starting point. Re-check official docs and current user/admin trend evidence before applying optimization claims.

## Local Baseline

- Host: MacBook Pro `Mac16,5`, Apple M4 Max, 16 CPU cores, 40 GPU cores, 48 GB unified memory.
- Parallels: Desktop 26.4.0-57513, license valid, CLI installed at `/usr/local/bin/prlctl` and `/usr/local/bin/prlsrvctl`.
- VM: `Windows 11`, `/Users/jstar/Parallels/Windows 11.pvm`.
- VM config observed: 4 vCPU, 8192 MB RAM, Apple hypervisor, Parallels Tools 26.4.0 installed, shared networking, IP `10.211.55.3`.
- Boundary config observed: host shared folders off, shared profile off, shared cloud off, guest-to-host app sharing off, host-to-guest app sharing on, shared clipboard on, printer sync on, host location sharing on, camera/microphone blocked by macOS privacy.
- Access test: Computer Use saw the Parallels window and Windows desktop, but Windows UI was pixels rather than native accessibility elements.
- Command test: `prlctl exec 'Windows 11' --current-user cmd /c whoami` returned `jayfreemanbdc2\jstar`.
- Idle auto-pause blocked active guest probes until disabled with `prlctl set 'Windows 11' --pause-idle off`.
- Large encoded PowerShell batches hung through `prlctl exec`; plain `cmd /c`, `reg`, `sc`, `schtasks`, and short PowerShell probes were reliable.

## Current-Source Findings

- Parallels documents `prlctl` and `prlsrvctl` as current CLI utilities for Pro and Business/Enterprise editions.
- Parallels recommends default/Automatic CPU and memory allocation for Windows 10 or later, then workload-specific manual tuning only if performance is unsatisfactory.
- Parallels warns that more CPU/RAM is not always better; older performance guidance says half of total CPUs is usually the upper useful range and exceeding it can degrade host and guest performance.
- Microsoft and Parallels both frame Parallels Desktop as an authorized path for Windows 11 Pro/Enterprise ARM on Apple Silicon, while Microsoft still documents limitations around nested virtualization, drivers, DirectX/gaming, and hardware-dependent software.
- Parallels 26 release notes include multiple Windows 11 and Coherence fixes, including GPU spike, window stacking, Excel/Outlook, OpenGL, crashes/freezes, and DNS issues.

## Trend Findings

- 4 vCPU and 8-12 GB RAM is a common productivity baseline on Apple Silicon; larger allocations should be justified by measured Windows-side memory pressure.
- Coherence improves convenience but has enough current issue history to treat as invited complexity for MSP work; prefer windowed/fullscreen until this exact workload proves Coherence is stable and efficient.
- Disk growth is usually snapshots, suspend state, Windows updates/temp files, and expanding virtual disks. Use Windows storage cleanup, remove stale snapshots, shut down the VM, then reclaim/compact.
- Keep Microsoft Defender on unless replacing it with a managed ARM-compatible EDR and explicit policy.
- Validate MSP tools by category. VPN, EDR, printer/scanner drivers, RMM agents, and older line-of-business apps are higher risk than Office, browsers, Teams, and PowerShell.

## First Optimization Pass Applied

- Parallels: limited clipboard to host-to-guest, disabled host printer sync, host-to-guest app sharing, guest notifications, Dock bounce alerts, host location sharing, and idle auto-pause for the test goal.
- Windows: disabled telemetry, geolocation, Xbox game services, Game DVR policy, Windows consumer/suggestion policies, Windows feeds/widgets policy, Delivery Optimization peer mode, and selected media/game scheduled tasks.
- Preserved: Defender, Windows Update, Windows Search, Microsoft Store/App Installer, Terminal, Quick Assist, Edge/WebView runtime, Office/Outlook/Teams packages, Parallels Tools, shared networking, and current CPU/RAM allocation.
- Deferred: optional feature removal and broad app package removal because encoded PowerShell batching hung and the fresh VM did not show enough resource pressure to justify deeper debloat yet.
